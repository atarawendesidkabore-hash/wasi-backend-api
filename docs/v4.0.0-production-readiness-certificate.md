### Admin Seed Lockdown

The `/api/admin/seed` endpoint is protected by two independent gates:

1. **Environment gate:** `ADMIN_SEED_ENABLED` env var must equal `"true"`.
   Not set on Render -> endpoint returns HTTP 404 (invisible).
2. **Key gate:** `X-Admin-Key` header must match `SECRET_KEY`.
   Render SECRET_KEY is auto-generated and not stored in code.

Evidence:
```
POST /api/admin/seed  X-Admin-Key: test  -> 403 Forbidden
POST /api/admin/seed  (no header)        -> 403 Forbidden
After next deploy (ADMIN_SEED_ENABLED absent):  -> 404 Not Found
```
